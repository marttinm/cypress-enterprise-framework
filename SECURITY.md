# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability, please email security@cypress-enterprise.local instead of using the issue tracker.

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will:
- Acknowledge receipt within 48 hours
- Provide updates every 5 days
- Work toward a fix and release
- Credit you in the security advisory (unless you prefer anonymity)

## Security Practices

### Our Commitment
- Keep dependencies up to date
- Review code for security issues
- Use branch protection rules
- Require code signing
- Scan for secrets and vulnerabilities
- Monitor for security alerts

### Your Responsibility
- Don't commit secrets (API keys, tokens, passwords)
- Review code for hardcoded sensitive data
- Keep your local environment secure
- Report vulnerabilities responsibly
